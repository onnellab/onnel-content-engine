# Image Guide

## ONNELLAB Content Engine

---

# 1. Purpose

This document defines the visual standards for every image produced by the ONNELLAB Content Engine.

The purpose is to ensure that every infographic, illustration, comparison, and diagram shares a consistent visual identity.

Images should support understanding.

They should never distract from it.

---

# 2. Primary Objective

Every image should answer one question.

A reader should understand the image within five seconds.

If the image requires lengthy explanation, it should be redesigned.

---

# 3. Image Philosophy

Images exist to explain.

Not to decorate.

Every visual should reduce cognitive effort.

The image should make the accompanying text easier to understand.

---

# 4. Visual Tone

Every image should feel:

* calm
* clean
* minimal
* structured
* approachable
* timeless

Avoid dramatic presentation.

Avoid excessive visual effects.

---

# 5. ONNELLAB Design Language

Every infographic should follow the ONNELLAB design system.

Preferred colors:

Background

* Ivory Soft

Primary

* Baby Blue Calm

Secondary

* Dusty Lilac

Accent

* Soft Peach

Text

* Charcoal Soft

High saturation should be avoided.

---

# 6. Typography

Use ONNEL Sans when available.

Until then:

* System UI fonts
* Pretendard (Korean)
* Inter (English)

Typography should remain secondary to the information.

Avoid decorative fonts.

---

# 7. Layout Principles

Every image should follow a simple hierarchy.

Preferred structure:

```text id="layoutimg"
Title

↓

Main Diagram

↓

Supporting Labels

↓

Short Notes
```

The diagram should occupy most of the visual space.

---

# 8. Image Categories

The Content Engine uses five standard image types.

## Workflow

Explains a sequence of actions.

Example:

```text id="workflowimg"
Problem

↓

Process

↓

Result
```

---

## Comparison

Compares two or more approaches.

Example:

```text id="compareimg"
Method A

VS

Method B
```

---

## Before / After

Shows improvement.

Example:

```text id="beforeafter"
Before

↓

After
```

---

## Architecture

Explains relationships between components.

Example:

```text id="archimg"
Input

↓

Processing

↓

Output
```

---

## UI Highlight

Explains an application feature.

Only highlight the relevant area.

Avoid showing the entire interface unless necessary.

---

# 9. Icons

Icons should be:

* simple
* line-based
* minimal
* consistent

Avoid:

* realistic illustrations
* 3D icons
* emoji as primary icons

Icons should support the explanation.

---

# 10. Screenshots

Application screenshots should be used only when they demonstrate a real feature.

Never use screenshots as decorative elements.

When possible:

* crop unnecessary UI
* highlight relevant controls
* maintain high resolution

---

# 11. Infographics

Infographics should:

* simplify complex ideas
* use very little text
* rely on visual structure

Avoid creating poster-style graphics.

Information always comes before appearance.

---

# 12. Motion

Animations are optional.

When used:

* subtle fade
* simple transitions
* minimal movement

Avoid:

* bouncing
* spinning
* excessive motion

Motion should support comprehension.

---

# 13. Branding

ONNELLAB branding should remain subtle.

Small logo placement is acceptable.

Large promotional branding is discouraged.

Readers should remember the explanation before the logo.

---

# 14. Accessibility

Images should remain understandable when:

* viewed on mobile devices
* scaled down
* printed
* viewed in grayscale

Color should never be the only way information is communicated.

---

# 15. Language Variants

Images that contain readable text must be generated separately for each published language.

Supported public article languages are:

* English
* Korean

Do not reuse an English text infographic in a Korean article.

Do not reuse a Korean text infographic in an English article.

Language-specific blog assets should use language-specific paths:

```text
/blog-assets/en/{slug}/...
/blog-assets/ko/{slug}/...
```

Screenshots without embedded explanatory text may be reused only when the UI itself is the subject and the reuse does not reduce comprehension.

Brand labels must remain in their official form.

Use:

```text
ONNELLAB Blog
```

Do not translate or localize the `ONNELLAB` brand name inside image assets.

---

# 16. AI Image Generation

When images are generated using AI:

* prioritize clarity
* avoid photorealistic marketing art
* maintain ONNELLAB visual language
* keep diagrams simple
* avoid unnecessary decorative details

The image should explain.

Not impress.

---

# 17. Quality Checklist

Before publishing, confirm that the image:

* explains one idea
* uses consistent colors
* follows the ONNELLAB style
* remains readable on mobile
* avoids unnecessary decoration
* supports the accompanying article
* matches the article language when the image contains text

If any item fails, revise the image.

---

# 18. Success Criteria

A successful image allows readers to understand the core concept almost immediately.

Readers should think:

"I understand."

before thinking:

"That looks nice."

---

# Final Principle

Design for understanding.

Beauty follows clarity.

# SEO Guide

## ONNELLAB Content Engine

---

# 1. Purpose

This document defines the Search Engine Optimization (SEO) principles for all ONNELLAB content.

Its purpose is to maximize long-term discoverability through high-quality, user-focused content rather than search engine manipulation.

This document applies to every article published by the ONNELLAB Content Engine.

---

# 2. SEO Philosophy

SEO exists to help users discover useful knowledge.

It does not exist to manipulate rankings.

Every optimization should improve the reader's experience.

Never optimize for algorithms at the expense of humans.

---

# 3. Search Intent First

Every article must begin with a clearly identified search intent.

Examples:

* How to...
* Why does...
* Best way to...
* Difference between...
* Can I...
* What is...

The article should completely satisfy that intent.

---

# 4. One Primary Topic

Each article should focus on one primary problem.

Do not attempt to rank for dozens of unrelated keywords.

One article.

One question.

One complete answer.

---

# 5. Title Rules

Titles should:

* clearly describe the problem
* contain the primary search phrase naturally
* avoid clickbait
* remain understandable without additional context

Preferred:

* How to Read Large TXT Files Without Lag
* Best Offline Music Player for FLAC Collections

Avoid:

* You Won't Believe This...
* The Ultimate Secret...
* Amazing Trick...

---

# 6. Heading Structure

Use a logical heading hierarchy.

Recommended structure:

```text id="seohead"
H1

H2

H2

H3

H2

Conclusion
```

Each section should answer one part of the reader's question.

---

# 7. Keyword Usage

Primary keywords should appear naturally in:

* title
* introduction
* headings (when appropriate)
* conclusion

Never force repetition.

Keyword stuffing is prohibited.

---

# 8. Semantic Coverage

Instead of repeating the same keyword, explain related concepts.

Example:

Primary topic:

TXT Reader

Related concepts:

* plain text
* large text files
* UTF-8
* encoding
* bookmarks
* search
* virtual rendering

A complete explanation is more valuable than repeated keywords.

---

# 9. Internal Linking

Every article should include links to:

* related articles
* related applications
* official documentation

Links must improve navigation.

Internal links should always make sense in context.

---

# 10. External References

When appropriate, reference:

* official specifications
* platform documentation
* recognized standards

Avoid relying on low-quality sources.

External links should increase credibility.

---

# 11. Images

Every article should include meaningful visuals.

Preferred image types:

* workflow diagrams
* comparison tables
* before / after examples
* UI screenshots
* process illustrations

Images should support the text rather than replace it.

---

# 12. Metadata

Every article should include:

* unique title
* meta description
* canonical URL
* publication date
* updated date
* category
* tags

Metadata should accurately describe the content.

---

# 13. URL Structure

URLs should be:

* short
* descriptive
* stable

Preferred:

```text id="seourl"
blog/read-large-txt-files
```

Avoid:

```text id="seobad"
blog/post?id=12345
```

---

# 14. Evergreen Priority

Whenever possible, create evergreen content.

Evergreen articles should remain useful for years.

Product announcements and release notes are exceptions.

---

# 15. Content Freshness

Evergreen articles should be reviewed periodically.

Possible updates include:

* screenshots
* feature additions
* improved explanations
* updated references

The article should evolve without changing its core purpose.

---

# 16. Product Mentions

Applications should only appear when they naturally solve the discussed problem.

Readers should never feel that the article exists solely to promote a product.

The educational content must remain valuable without any product recommendation.

---

# 17. Performance

Pages should prioritize:

* fast loading
* responsive layout
* accessible typography
* readable formatting

Technical quality supports search quality.

---

# 18. Success Criteria

A successful article:

* satisfies search intent
* answers the question completely
* remains useful over time
* earns natural backlinks
* encourages readers to continue exploring ONNELLAB resources

Ranking should be the result of quality.

Never the primary objective.

---

# Final Principle

Optimize for understanding.

Search engines will follow.
